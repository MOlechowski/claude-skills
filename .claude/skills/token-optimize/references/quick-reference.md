# Token Optimization Checklist

## Remove

- [ ] Filler: just, very, really, basically, actually, simply
- [ ] Hedging: might, possibly, perhaps, maybe
- [ ] Redundant adjectives: completely, totally, absolutely
- [ ] Empty phrases: it should be noted that, it is important to

## Replace

| Verbose | Concise |
|---------|---------|
| in order to | to |
| due to the fact that | because |
| at this point in time | now |
| is able to | can |
| in the event that | if |
| for the purpose of | for |
| with regard to | about |
| a large number of | many |
| the majority of | most |
| in spite of the fact that | although |
| on a daily basis | daily |
| at the present time | now |
| in close proximity to | near |
| has the ability to | can |
| make a decision | decide |
| take into consideration | consider |
| come to the conclusion | conclude |

## Transform

- Passive → Active voice
- Multiple sentences → Combined when related
- Nested lists → Flat when possible
- Long paragraphs → Short ones

## Preserve

- YAML frontmatter (---...---)
- Code blocks (fenced and inline)
- URLs and file paths
- Technical terms
- Heading structure
- List semantics
